# Community Enumeration

## Overview

Default communities (public, private) and guessable strings grant read access to system data.

## Commands

Discover SNMP service:
    nmap -sU -p161 --script=snmp-info <target IP>

Brute force from host list:
    onesixtyone -c /usr/share/doc/onesixtyone/dict.txt -i 161.txt
    onesixtyone -c /usr/share/seclists/Discovery/SNMP/snmp-onesixtyone.txt -i targets.txt

Quick check (defaults to public if no -c):
    snmp-check -v2c -c public <target IP>

Manual probe:
    snmpwalk -v2c -c public <target IP> 1.3.6.1.2.1.1.1.0
    snmpwalk -v1 -c public <target IP> 1.3.6.1.2.1.1.1.0

## Validation / False Positives

Timeout or no response is not proof SNMP is disabled — verify UDP/161 is reachable and try v1 and v2c.
A single failed community does not rule out a custom string — use a targeted wordlist.


# MIB Walk

## Overview

Standard MIB OIDs expose OS version, interfaces, routes, processes, and installed software.

## Commands

System and host:
    1.3.6.1.2.1.1.1.0        sysDescr (hardware, OS, model)
    1.3.6.1.2.1.1.2.0        sysObjectID
    1.3.6.1.2.1.1.3.0        sysUpTime
    1.3.6.1.2.1.1.4.0        sysContact
    1.3.6.1.2.1.1.5.0        sysName (hostname)
    1.3.6.1.2.1.1.6.0        sysLocation

Interfaces:
    1.3.6.1.2.1.2.2.1        ifTable
    1.3.6.1.2.1.2.2.1.2      ifDescr (e.g., eth0)
    1.3.6.1.2.1.2.2.1.6      ifPhysAddress (MAC)
    1.3.6.1.2.1.2.2.1.8      ifOperStatus (up/down)

Host resources:
    1.3.6.1.2.1.25.1.1.0     hrSystemUptime
    1.3.6.1.2.1.25.2.3       hrStorage
    1.3.6.1.2.1.25.3.2       hrDevice
    1.3.6.1.2.1.25.4.2       hrSWRun (processes)
    1.3.6.1.2.1.25.6.3       hrSWInstalled

Network:
    1.3.6.1.2.1.4.20         ipAddrTable
    1.3.6.1.2.1.4.22         ipRouteTable
    1.3.6.1.2.1.6.13         tcpConnTable
    1.3.6.1.2.1.7.5          udpTable

SNMP stats:
    1.3.6.1.2.1.11.1.0       snmpInPkts
    1.3.6.1.2.1.11.28.0      snmpSilentDrops (failed auth)

Walk a branch:
    snmpwalk -v2c -c <community string> <target IP> <MIB>

## Validation / False Positives

Partial walks on busy devices may truncate — increase timeout or walk specific OIDs.
sysDescr strings are informational — correlate with CVE databases before claiming exploitability.


# Read-Write Testing

## Overview

RW community strings allow configuration changes. Test only with approval using harmless OIDs.

## Commands

Attempt a benign write (sysLocation):
    snmpset -v2c -c <community string> <target IP> 1.3.6.1.2.1.1.6.0 s "pentest-marker"

Verify:
    snmpwalk -v2c -c <community string> <target IP> 1.3.6.1.2.1.1.6.0

## Validation / False Positives

snmpset returning success on a read-only community may be misleading — re-walk the OID to confirm persistence.
Restore any changed values after testing.


# Post-Enumeration

## Overview

Turn SNMP output into follow-on targets — version strings for CVEs, interfaces for VLAN mapping, routes for internal ranges.

## Payloads / Techniques

Correlate sysDescr/sysObjectID with IOS/IOS-XE or vendor firmware CVEs (older Cisco 12.x and early 16.x are high value).
Map interface and VLAN data, then scan exposed management services (SSH, HTTPS, Telnet) on adjacent subnets.
Use process and software tables to identify backup agents, AV, or domain-joined services.

## Validation / False Positives

SNMP-reported software versions may lag actual installed patches — treat as a lead, not proof of exploitability.
