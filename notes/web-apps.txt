# Example Assessment

## Commands

- Set variables
URL=target.com
IP=$(dig +short $URL)
SECLISTS=/usr/share/wordlists/seclists/Discovery/Web-Content
AGENT='Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/150.0.0.0 Safari/537.36 Edg/150.0.4078.48'

- Test connectivity
ping -c 3 $URL
traceroute $URL

- Discover open ports and perform service enumeration
nmap -Pn -n -T4 --open -sTVC --stats-every 20s $URL -oA nmap

- Enumerate web technologies
whatweb -a 3 -U="$AGENT" https://$URL
while read -r i; do whatweb -a 3 -U="$AGENT" "https://$i/" > "$i.txt" 2>&1; echo "Scanning $i"; done < 443.txt

cat > headers.txt <<EOF
User-Agent: $AGENT
Accept: text/html,application/xhtml+xml,application/xml;q=0.9,image/avif,image/webp,*/*;q=0.8
Accept-Language: en-US,en;q=0.5
Accept-Encoding: gzip, deflate, br
Connection: keep-alive
Upgrade-Insecure-Requests: 1
EOF

wafw00f -H headers.txt -a -i 443.txt

- SSL/TLS enumeration
See ssl.txt for sslscan, sslyze, nmap scripts, and certificate checks.
Discover option: web/ssl.sh (reads IP:port list from 443.txt).

- Default configurations and files
nikto -h "https://$URL" -Format htm --output nikto.htm

feroxbuster -u "https://$URL" -a "$AGENT" --auto-tune -C 404,502,503 --no-recursion \
     -w "$SECLISTS/directory-list-2.3-small.txt" --output feroxbuster.txt
Or replace -C <CODES> with -s 200

- Directory enumeration
ffuf -u "https://$URL/FUZZ" -mc 200,204,301,302,307,401,403 -fc 404 -c -r -t 40 \
     -H "User-Agent: $AGENT" -recursion -recursion-depth 2 \
     -e .php,.asp,.aspx,.jsp,.html,.js,.json,.bak,.old,.txt,.log \
     -w "$SECLISTS/directory-list-2.3-medium.txt"

- Discover vulnerabilities
nuclei -update-templates
nuclei -u "https://$URL" -tags cve,exposure,misconfig,vuln -c 10 -rl 50

- API and schema discovery
ffuf -u "https://$URL/FUZZ" -w "$SECLISTS/raft-large-words.txt" -mc 200,301,302,401,403 \
     -H "User-Agent: $AGENT" -t 40
Common paths: /api, /api/v1, /swagger, /swagger.json, /openapi.json, /graphql, /graphiql, /docs

- Application specific enumeration
davtest -url "https://$URL"
droopescan scan drupal -u "https://$URL"
joomscan -u "https://$URL"
uniscan -u "https://$URL" -qw
wpscan --url "https://$URL" --enumerate vp,vt,cb,u,m

## Validation / False Positives

Review nikto and nuclei output in Repeater before reporting — see web-burp.txt.
A 403 on /api or /swagger still confirms the path exists; note it for manual review.


# HTTP Methods

GET       Used to get data from a server. Passes all parameters in the URL
POST      Used to send data to a server. Passes all parameters in the body of the request
HEAD      The web app will only return the response headers, not the body
OPTIONS   Show a list of HTTP options that the server supports
CONNECT   Set up a tunnel between a client and a server for SSL encryption or an HTTP proxy
TRACE     Used for debugging
TRACK     Returns the full HTTP request back to the requesting client for proxy-debugging
PUT       Used for adding content to a web app, common with RESTful APIs
PATCH     Used for partial updates, common with RESTful APIs
DELETE    Used for deleting content from a web app, common with RESTful APIs

# Fuzzing

When fuzzing headers, disable URL Encode.
When you are inside of a POST parameter, always use URL encoding.

# Open Redirect

## Overview

Quick workflow for redirect parameters. See open-redirect.txt for parameters and payloads.

## Commands

Burp > Proxy > find parameters: redirect, return, next, url, goto, continue
Repeater > set value to https://<attacker domain> or //<attacker domain>
Follow 302 chain to confirm external landing.

## Validation / False Positives

See web-vulns.txt and web-burp.txt. Redirect to /login or same-site paths is not an open redirect.


# GraphQL and REST APIs

## Overview

API testing after discovery of /api, /graphql, /swagger, or JSON endpoints.

## Commands

- REST smoke test
curl -ik "https://$URL/api/v1/users" -H "Authorization: Bearer <token>"
curl -ik -X POST "https://$URL/api/v1/users" -H "Content-Type: application/json" -d '{"user":"test"}'

- GraphQL introspection
curl -ik -X POST "https://$URL/graphql" -H "Content-Type: application/json" -d \
'{"query":"{ __schema { types { name } } }"}'

- GraphQL batching / depth (manual check in Repeater)
Send array of queries or deeply nested query to test rate limits and authorization.

- OpenAPI / Swagger
Browse /swagger, /swagger-ui, /openapi.json for hidden endpoints and parameters.

## Validation / False Positives

Introspection enabled is informational unless sensitive types or fields are exposed.
A 401 on an API route confirms existence — pair with IDOR and auth tests in web-auth.txt.
