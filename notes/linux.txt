# SSH

## Overview

Keys, port forwarding, SOCKS, and X11.

## Commands

    ssh-keygen                                      Generate key pair
    ssh-copy-id <user>@<host>                       Copy pubkey to authorized_keys

    vi /etc/ssh/sshd_config                         Change Port 22 if needed
    systemctl start ssh
    systemctl enable ssh

Permissions:
    -rw-------  authorized_keys
    -rw-------  id_rsa
    -rw-r--r--  id_rsa.pub

ssh <user>@<host>                                       Standard connection
ssh <user>@<host> -p 2222                               Non-standard port
ssh <user>@<host> -i <key>                              Identity file

Local forward:
    ssh <user>@<host> -L <local port>:<remote host>:<remote port>

Remote forward:
    ssh <user>@<host> -R 0.0.0.0:22:127.0.0.1:22

Dynamic forward (SOCKS):
    ssh <user>@<host> -f -N -D 1080
    proxychains nmap <options>

X11:
    # ForwardX11 yes in ssh_config; X11Forwarding yes in sshd_config
    ssh -X <user>@<host> -o "XAuthLocation=/opt/X11/bin/xauth"
    export DISPLAY=:0

Legacy KEX:
    ssh -o KexAlgorithms=diffie-hellman-group-exchange-sha256 <user>@<target>

## Validation / False Positives

Remote forward requires GatewayPorts or bind setup on the server; connection refused often means sshd config.
chmod 600 on private keys is mandatory; ssh rejects looser permissions.
