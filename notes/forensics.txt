# Drive Naming

## IDE (legacy)

hda         primary master IDE
hdb         primary slave IDE
hdc         secondary master IDE
hdd         secondary slave IDE


# SCSI USB SATA Naming

sda         1st
sdb         2nd
sde         5th
sdz         26th


# Partition Numbering

## Overview

1–4          partitions, primary or extended
5 and up     logical drives

Examples:
    hda3       primary IDE master, 3rd partition
    hdd4       secondary IDE slave, 4th partition
    hdc5       secondary IDE master, 1st logical partition
    sdb9       second SCSI device, 5th logical partition

## Commands

bt nohd                      mount NTFS read-only at BackTrack/Kali boot
fdisk -l                     show disk and partition info


# Imaging with dd

## Overview

Boot read-only, image source disk to USB destination in split chunks. Legacy single-IDE example uses hda1 → sda1.

## Commands

1. Enter BIOS — CD/DVD first in boot order
2. Boot BackTrack/Kali with bt nohd at boot prompt
3. Identify source:
    fdisk -l
4. Mount source read-only:
    cd /mnt
    mkdir hda1
    mount -o ro /dev/hda1 /mnt/hda1
5. Plug in USB destination drive
6. Identify destination:
    fdisk -l
7. Mount destination read/write:
    cd /mnt
    mkdir sda1
    mount -o rw /dev/sda1 /mnt/sda1
    mount
8. Image to split files:
    cd /mnt/sda1
    mkdir backup && cd backup
    dd if=/dev/hda bs=512 conv=sync,noerror | split -b 640m - hd.img


# Autopsy

## Overview

View dd image in Autopsy browser UI. Menu paths are legacy BackTrack — on Kali use autopsy package or Sleuth Kit equivalent.

## Commands

1. Connect external drive with dd image
2. Launch Autopsy (legacy: K > Backtrack > Digital Forensics > Forensic Analysis > Autopsy)
3. Firefox > http://localhost:9999/autopsy
4. Open Case > New Case — enter description and investigator
5. New Case > Add Host — enter host info
6. Add Host > Add Image — e.g. /mnt/sda1/BackTrack/hd.img*
7. Select Disk or Partition > Default import: Symlink > Next
8. Verify all parts > Next
9. Calculate image hash
10. Under partition 1, note mount point and filesystem > Add > OK
11. C:/ > Analyze > File Analysis


# pasco index.dat

## Overview

Parse Internet Explorer index.dat URL history from mounted image paths.

## Commands

    find / -name index.dat

    pasco "/mnt/hda1/Documents and Settings/Administrator/UserData/index.dat"
    pasco -t , "/mnt/hda1/Documents and Settings/Administrator/UserData/index.dat"
    pasco -d "/mnt/hda1/Documents and Settings/Administrator/UserData/index.dat"
    pasco "/mnt/hda1/Documents and Settings/Administrator/UserData/index.dat" > browser.txt


# vinetto Thumbs.db

## Overview

Extract thumbnails from Windows Thumbs.db. Legacy menu: K > Backtrack > Digital Forensics > Forensic Analysis > Vinetto.

## Commands

    find / -name Thumbs.db

    vinetto -o "/mnt/hda1/Documents and Settings/Administrator/My Documents/My Pictures/Thumbs.db"

    mkdir /mnt/sda1/thumbs
    vinetto -o /mnt/sda1/thumbs "/mnt/hda1/Documents and Settings/Administrator/My Documents/My Pictures/Thumbs.db"

HTML report with extracted pictures:
    vinetto -oH /mnt/sda1/thumbs "/mnt/hda1/Documents and Settings/Administrator/My Documents/My Pictures/Thumbs.db"


# magicrescue JPEG Recovery

## Overview

Carve JPEG files from a USB device using predefined recipes.

## Commands

    cd /usr/local/share/magicrescue/recipes
    ls

Insert USB thumb drive:
    fdisk -l
    mkdir foundfiles

    magicrescue -r jpeg-jfif foundfiles /dev/sda1
    cd foundfiles && ls

View recovered image:
    kolorpaint <file>.jpg


# Validation / False Positives

Always mount evidence read-only (bt nohd or mount -o ro) before imaging — writes destroy forensic integrity.
fdisk -l device names vary by machine (NVMe /dev/nvme0n1, USB /dev/sdb1) — verify before dd, not assumed hda1/sda1.
split -b 640m requires lowercase m on modern coreutils — 640 m was legacy split syntax.
Autopsy, pasco, and vinetto menu paths target BackTrack — on current Kali confirm package install and paths.
magicrescue output is carved data, not file-system metadata — false JPEG headers are common; verify with file or viewer.
