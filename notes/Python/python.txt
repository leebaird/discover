# Virtual Environments

## Overview

Kali marks the system Python as externally managed (PEP 668). Install third-party packages inside a venv, not with system-wide pip.

## Commands

Set up a cloned tool:
    git clone https://github.com/some/new/python/tool
    cd <new tool>/
    python3 -m venv venv
    source venv/bin/activate
    pip install <dependency>
    pip install -r requirements.txt

Named venv outside the repo:
    python3 -m venv ~/venvs/<tool-name>
    source ~/venvs/<tool-name>/bin/activate

Run a script with a venv interpreter without activating:
    ~/venvs/<tool-name>/bin/python script.py

## Validation / False Positives

pip install outside a venv fails with externally-managed-environment; create and activate venv first.
Forgotten deactivate leaves the wrong python on PATH; run which python3 to confirm.


# argparse

## Overview

Preferred CLI parsing for companion scripts. See fuzzer-pop3.py and test.py.

## Commands

    import argparse

    parser = argparse.ArgumentParser(description='Connect to a host and port')
    parser.add_argument('ip', help='target IP address')
    parser.add_argument('port', type=int, help='target port')
    parser.add_argument('-v', '--verbose', action='store_true', help='verbose output')
    args = parser.parse_args()

    print(args.ip, args.port)

## Validation / False Positives

type=int raises on non-numeric port; argparse prints usage and exits automatically.
Positional vs optional argument order matters; put positional args before flags in the script definition.


# Script Skeleton

## Overview

Interactive port-validation pattern. Runnable copy: ex1.py.

## Commands

    #!/usr/bin/env python3

    import os
    import sys

    def main():
        os.system('clear')
        port = input('\nEnter a valid port: ')

        if port == '':
            print('\nYou did not enter anything.\n')
            sys.exit(1)

        try:
            val = int(port)
        except ValueError:
            print('\nThat is not a number.\n')
            sys.exit(1)

        if not 1 <= val <= 65535:
            print('\nThat is an invalid port.\n')
            sys.exit(1)

        print('\nThat is a valid port.\n')

    if __name__ == '__main__':
        main()


# File I/O

## Overview

Read and write text files. Prefer context managers so files close on error.

## Commands

Read:
    with open('tmp', 'r') as f:                          Read a file
        filedata = f.read().split('\n')                  Split into a list of lines

Write:
    with open('tmp', 'w') as f:                          Write to a file
        f.write('These are my notes.')

Append:
    with open('tmp', 'a') as f:                          Append to a file
        f.write('more data\n')

Legacy style (avoid in new scripts):
    f = open('tmp', 'r')                                 Setup a read connection to a file
    filedata = f.read()
    f.close()

## Validation / False Positives

open() without encoding assumes locale-dependent text mode; use encoding='utf-8' for portable scripts.
Forgetting f.close() on legacy paths leaks handles; use with open(...) instead.
Binary mode ('rb'/'wb') is required for non-text payloads such as shellcode files.

# Subprocess

## Overview

Run external programs. Prefer argument lists over shell=True unless shell features are required.

## Commands

    import subprocess

    subprocess.run(['python3', 'recon.py'], check=True)              Run a Python script
    subprocess.run(['./script.sh'], check=True)                     Run a script directly
    result = subprocess.run(['id'], capture_output=True, text=True)  Capture stdout/stderr
    print(result.stdout)

    subprocess.run('./script.sh | grep FOO', shell=True)            Shell pipeline (injection risk)

## Validation / False Positives

check=True raises CalledProcessError on non-zero exit; handle or omit check for expected failures.
shell=True with user-controlled input enables command injection; pass a list when possible.
capture_output=True buffers all output; unsuitable for long-running streaming commands.

# HTTP Client

## Overview

HTTP requests from Python. Install requests inside a venv: pip install requests.

## Commands

    import requests

    r = requests.get('https://<target>/api', timeout=10)
    r = requests.get('https://<target>/api', verify=False, timeout=10)   Skip TLS verify (lab only)
    print(r.status_code, r.text)

    r = requests.post('https://<target>/login', data={'user': '<user>', 'pass': '<password>'})

## Validation / False Positives

verify=False silences TLS warnings but hides MITM; use only in controlled lab traffic.
Missing timeout hangs on unresponsive hosts; always set timeout=.
Non-2xx status codes do not raise by default; check r.status_code or r.raise_for_status().


# IP Address Sorting

## Overview

Read IPs from a file, dedupe, sort numerically with ipaddress. Skips invalid lines.

## Commands

    import ipaddress

    ips = set()
    with open('file.txt', 'r') as fh:
        for line in fh:
            line = line.strip()
            if not line:
                continue
            try:
                ips.add(ipaddress.ip_address(line))
            except ValueError:
                pass

    print('\n'.join(map(str, sorted(ips))))

## Validation / False Positives

CIDR notation (10.0.0.0/24) requires ip_network(), not ip_address().
Hostnames in the input file are skipped silently by the try/except loop.

# Host List Parsing

## Overview

Normalize scraped hostnames: skip wildcards, leading digits, and bare dots.

## Commands

    def add_host(line, word, hosts):
        i = line.strip()
        # skip when word is empty or only 'www.'; skip wildcards and leading-digit hosts
        if (len(i) > 1
                and word.replace('www.', '')
                and not i.startswith('*')
                and not i.lstrip()[0].isdigit()):
            hosts.add(i[1:] if i[0] == '.' else i)
