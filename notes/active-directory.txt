# External Recon

## Overview

Enumerate users, groups, trusts, and AD-related DNS from Linux or a foothold without domain credentials.

## Commands

Domain users and groups (from domain-joined or credentialed host):
    net user /domain
    net user <user> /domain
    net group /domain
    net group "Domain Computers" /domain
    net group "Domain Admins" /domain
    net group "Enterprise Admins" /domain
    nltest /dclist:<domain>
    nltest /domain_trusts /all_trusts
    systeminfo

AdFind (from Windows):
    AdFind.exe -f "(objectcategory=person)" > ad_users.txt
    AdFind.exe -f "(objectcategory=computer)" > ad_computers.txt
    AdFind.exe -f "(objectcategory=organizationalUnit)" > ad_ous.txt
    AdFind.exe -f "(objectcategory=group)" > ad_group.txt
    AdFind.exe -sc trustdmp > trustdmp.txt
    AdFind.exe -gcb -sc trustdmp > trustdmp2.txt
    AdFind.exe -subnets -f "(objectCategory=subnet)" > subnets.txt

DNS SRV records:
    dig -t SRV _gc._tcp.<domain>
    dig -t SRV _ldap._tcp.<domain>
    dig -t SRV _kerberos._tcp.<domain>
    dig -t SRV _kpasswd._tcp.<domain>
    nmap --script dns-srv-enum --script-args "dns-srv-enum.domain='<domain>'"

Network discovery:
    nmap --script broadcast-dhcp-discover
    dnsrecon -r <class C> -n <DNS server>

Current domain (PowerShell on domain-joined host):
    [System.DirectoryServices.ActiveDirectory.Domain]::GetCurrentDomain()
    # Review PdcRoleOwner and Name

## Validation / False Positives

DNS SRV records confirm AD presence — still map which hosts are live before reporting full domain compromise.
AdFind output requires a domain-joined host or valid LDAP credentials in most environments.


# Domain Controller Identification

## Commands

nbtstat -a <target IP>
    # Hex code 1C next to the domain name indicates a domain controller

dsquery server
Get-NetDomainController
net group "domain controllers" /domain
net view \\<server>
cmd.exe /c set
    # LOGONSERVER variable

Specific domain:
    dsquery server -domain <domain.fqdn>
    Get-NetDomainController -domain <domain.fqdn>
    nslookup <domain.fqdn>
    nslookup
        set type=all
        _ldap._tcp.<domain.fqdn>

Forest-wide:
    dsquery server -forest
    Get-NetForestDomain

## Validation / False Positives

1C in nbtstat indicates DC role for that name — confirm with LDAP/SMB and port 88/389/636.


# On-Domain Enumeration

## Overview

Run from a domain-joined Windows host with PowerView or built-in dsquery. Import PowerView from /opt/PowerSploit/Recon/ or host at http://<attacker IP>/.

## Commands

Computers:
    dsquery computer
    dsquery * -filter "(objectclass=computer)" -attr dnshostname,description,operatingsystem,operatingsystemversion -limit 0
    Get-NetComputer
    Get-NetComputer -FullData | select dnshostname,description,operatingsystem,operatingsystemversion
    dsquery * -filter "(&(objectclass=computer)(operatingsystem=*server*))" -attr dnshostname,description,operatingsystem,operatingsystemversion -limit 0

Users:
    dsquery user -limit 0
    Get-NetUser
    Get-NetUser | select samaccountname
    dsquery * -filter "(&(objectclass=user)(samaccountname=<user>))" -attr *
    Get-NetUser -Identity <user>
    dsquery * -filter "(&(objectclass=user)(useraccountcontrol>=65535))" -attr samaccountname,name
    Get-NetUser -Filter "useraccountcontrol>=65535" | select samaccountname,name

SPNs (see ad-attacks.txt for Kerberoasting):
    dsquery * -filter "(serviceprincipalname=*)" -attr samaccountname,serviceprincipalname
    Get-NetUser -SPN | select samaccountname,serviceprincipalname
    setspn -T <domain> -Q MSSQLSvc/*

Administrators:
    dsquery * -filter "(&(objectclass=user)(admincount=1))" -attr samaccountname,name
    dsquery group -name *admin*
    Get-NetUser -AdminCount | select samaccountname,name
    Get-NetGroup -FullData -GroupName "*admin*" | select member
    dsquery * -filter "name=domain admins" -attr member -l
    Get-NetGroup -FullData -GroupName "domain admins" | select member
    net group "domain admins" /domain
    dsquery * -filter "name=enterprise admins" -attr member -d <forest.fqdn>
    net localgroup administrators /domain

OUs and GPOs:
    dsquery ou -limit 0
    Get-NetOU -FullData
    dsquery * "ou=<ou>,dc=<domain>,dc=<tld>" -filter "(objectclass=user)"
    dsquery * -filter "(objectclass=grouppolicycontainer)" -attr name,displayname
    Get-NetGPO | select name,displayname
    dsquery * -filter "(objectcategory=organizationalunit)" -attr distinguishedname,gplink

Trusts, sites, subnets:
    dsquery * -filter "(objectclass=TrustedDomain)" -attr trustpartner,flatname,trustdirection
    Get-NetDomainTrust
    dsquery subnet
    Get-NetSubnet
    dsquery site
    Get-NetSite
    dsquery partition

## Validation / False Positives

admincount=1 includes protected accounts — not every result is a exploitable admin path.
Stale computer objects still appear in LDAP — correlate with live host discovery.
