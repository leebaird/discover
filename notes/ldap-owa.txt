# LDAP Discovery

## Overview

Find LDAP services and test whether unauthenticated or weakly authenticated access is possible.

## Commands

ip=<target IP>
echo $ip

shodan host $ip

nmap -Pn -n -T4 --open $ip
nmap -Pn -n -T4 --open -p389,636,3269 --script=ldap*,ssl* $ip

Anonymous / null bind test:
    ldapsearch -x -H ldap://$ip -b "dc=target,dc=com" -s sub
    ldapsearch -x -H ldap://$ip:389 -b "dc=target,dc=com" "*" | awk '/^dn: / {print $2}'

Unauthenticated bind with DN and blank password:
    ldapsearch -x -D "cn=admin,dc=target,dc=com" -s sub "cn=*" -h $ip

## Validation / False Positives

Anonymous LDAP bind is disabled on most modern domain controllers — failure is expected, not proof of hardening unless other enum still fails.
An open port 389 confirms LDAP service — not proof of useful data without a successful bind.


# LDAP Enumeration

## Overview

Extract users, computers, and groups with valid credentials or tools that wrap LDAP.

## Commands

ldapsearch (authenticated):
    ldapsearch -x -h <IP> -D 'DOMAIN\username' -w '<password>' -b "CN=Users,DC=target,DC=com"
    ldapsearch -x -h <IP> -D 'DOMAIN\username' -w '<password>' -b "CN=Computers,DC=target,DC=com"
    ldapsearch -x -h <IP> -D 'DOMAIN\username' -w '<password>' -b "CN=Domain Admins,CN=Users,DC=target,DC=com"
    ldapsearch -x -h <IP> -D 'DOMAIN\username' -w '<password>' -b "CN=Administrators,CN=Builtin,DC=target,DC=com"
    ldapsearch -x -h <IP> -D 'DOMAIN\username' -w '<password>' -b "CN=Remote Desktop Users,CN=Builtin,DC=target,DC=com"

LDAPS:
    ldapsearch -x -H ldaps://<IP>:636 -D 'DOMAIN\username' -w '<password>' -b "DC=target,DC=com"

ldapdomaindump:
    ldapdomaindump -u 'domain\username' -p '<password>' --no-json --no-grep ldap://<IP>
    ldapdomaindump -u 'domain\username' -p '<password>' --no-json --no-grep ldaps://<IP>:636

ldeep:
    ldeep ldap -u 'domain\username' -p '<password>' -d <domain.fqdn> -s ldap://<IP> enum
    ldeep ldap -u 'domain\username' -p '<password>' -d <domain.fqdn> -s ldap://<IP> bloodhound

crackmapexec LDAP auth check:
    crackmapexec ldap <IP> -u <user> -p '<password>'
    crackmapexec ldap <IP> --port 636 -u <user> -p '<password>'

## Validation / False Positives

ldapsearch returning only built-in groups may mean limited bind rights — try Users and Computers bases separately.
ldapdomaindump output should be reviewed for stale disabled accounts before spraying.


# LDAP Password Spraying

## Overview

Try one password against many accounts. Watch lockout policy — coordinate spray volume with the customer.

## Commands

Single password against user list:
    crackmapexec ldap <IP> -u users.txt -p '<password>' --continue-on-success
    crackmapexec ldap <IP> --port 636 -u users.txt -p '<password>' --continue-on-success

Password list against one user (noisier):
    crackmapexec ldap <IP> -u <user> -p passwords.txt

User list sources:
    /usr/share/wordlists/seclists/Usernames/
    enum4linux-ng -A <IP>
    ldapdomaindump / ldeep output

## Validation / False Positives

LDAP auth success must be followed by SMB, WinRM, or OWA verification — LDAP-only success may be a low-priv service account.
Stop spraying after the agreed attempt count; one lockout event is a finding even if no creds are recovered.


# OWA and Exchange

## Overview

Enumerate and spray Outlook Web Access / Exchange endpoints when LDAP or mail services are exposed.

## Commands

Autodiscover:
    https://autodiscover.<domain>/autodiscover/autodiscover.xml
    https://<domain>/autodiscover/autodiscover.xml
    curl -ik "https://<domain>/autodiscover/autodiscover.xml"

SprayingToolkit recon:
    atomizer.py owa 'https://webmail.<domain>/Autodiscover' --recon
    atomizer.py owa 'https://webmail.<domain>/ews/exchange.asmx' --recon

MailSniper (PowerShell on Windows attack host):
    ipmo C:\Tools\MailSniper\MailSniper.ps1
    Invoke-DomainHarvestOWA -ExchHostname <mail server IP>
    Invoke-PasswordSprayOWA -ExchHostname mail.<domain> -UserList C:\users.txt -Password '<password>' -OutFile creds.txt
    GetGlobalAddressList -UserName domain\username -Password '<password>' -ExchHostname webmail.<domain>/ews/exchange.asmx -OutFile gal.txt

Mailbox / username discovery:
    Invoke-SelfSearch -Mailbox user@<domain> -ExchHostname webmail.<domain>

Additional tools:
    https://github.com/franc-pentest/ldeep
    https://github.com/codewatchorg/PowerSniper

## Validation / False Positives

Autodiscover XML confirms Exchange presence — not proof of weak credentials until spray or login succeeds.
OWA spray lockouts can affect mobile clients — document time window and account scope with the customer.


# GUI Enumeration

## Overview

Apache Directory Studio is optional when you already have valid LDAPS credentials and prefer GUI browsing.

## Commands

File > New > LDAP Browser > LDAP Connection
    Hostname: <target IP>
    Port: 636
    Encryption: SSL
    Authentication: Simple
    User: domain\user

Search example:
    Search base: DC=target,DC=com
    Filter: (cn=admin*)
    Returning attributes: cn, mail, description

## Validation / False Positives

GUI results should be exported or screenshotted for reporting — manual browsing alone is hard to reproduce in the report.
